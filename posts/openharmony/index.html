分布式访问文件遇到devsl permission denied

在服务器端创建了文件之后，客户端通过hmdfs分布式文件系统去访问该文件提示devsl permission denied。 客户端报错： [ 109.699430] hmdfs: hmdfs_do_open_remote() hmdfs_send_open return failed with -13 服务器端报错： [ 174.455917] hmdfs: ctrl_cmd_update_devsl_handler() Found peer: device_id = 1 [ 176.481693] hmdfs: hmdfs_node_call_evt_cb() node 0x1:0x1 call online async cb seq 2 [ 176.481744] hmdfs: hmdfs_stash_online_do_restore() peer 0x1:0x1 restore stashed-file ok 0 fail 0 keep 0 [ 189.562784] hmdfs: hmdfs_open_file() devsl:1 [ 189.563653] hmdfs: check_sec_level() value: [ 189.563699] hmdfs: check_sec_level() error: -61,devsl: 1 [ 189.563736] hmdfs: hmdfs_open_file() devsl permission denied 报错函数： static int check_sec_level(struct hmdfs_peer *node, const char *file_name) { …… err = vfs_getxattr(file_path.dentry, DATA_SEC_LEVEL_LABEL, value, value_len); hmdfs_info(“value: %s”, value); hmdfs_info(“error: %d,devsl: %u”, err, node->devsl); if (err <= 0 && node->devsl >= DATA_SEC_LEVEL3) goto out; if (err > 0 && node->devsl >= parse_data_sec_level(value, err)) goto out; …… }

  • 一种方法是

     1、kernel_talker.cpp: .devsl = DATA_SEC_LEVEL3,
 2、devsl_dispatcher.cpp levelInfo = DATA_SEC_LEVEL3

    但是该方法只能使得A端访问B端文件，B端访问A端文件仍会出现devsl permission denied的问题

  • 提高设备的设备等级，参考https://gitee.com/zkx48/dfs_service_instruction

单独编译openharmony内核或者是增加内核模块

由于openharmony会在标准linux内核上打patch，不能直接在工作目录的linux_5.10下增加内核模块，必须在out目录下操作才可

单独编译的命令如下

export OHOS_ROOT=~/Documents/DevEco/Projects/RK3568
export PATH=$OHOS_ROOT/prebuilts/clang/ohos/linux-x86_64/llvm/bin:$PATH
export PATH=$OHOS_ROOT/prebuilts/gcc/linux-x86/arm/gcc-linaro-7.5.0-arm-linux-gnueabi/bin:$PATH
export MAKE_OPTIONS="ARCH=arm CROSS_COMPILE=arm-linux-gnueabi- CC=clang HOSTCC=clang"
export PRODUCT_PATH=vendor/hihope/RK3568
./make-ohos.sh TB-RK3568X0

./build.sh –product-name rk3568 | tee build.log
